# 3D GenAI 六部曲 git 归位 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把已在线上的 3D GenAI 六部曲在 git 里正确归位:大小写归一、纳入新内容、删脚手架、pngquant 压图。

**Architecture:** 纯索引操作完成大小写切换(`git rm --cached` 大写 + `git add` 小写,不触碰磁盘 → 零 checkout 风险);先删脚手架与压图,再做索引归一,使最终入库的小写树即清理压缩后的状态。全程在 `feature/3dgenai` 分支。

**Tech Stack:** git、pngquant(Homebrew)、Hugo extended、macOS sips(尺寸核对)。

## Global Constraints

- 分支隔离:全程在 `feature/3dgenai`,不直接推 `main`(spec 决策)
- 大小写命令严格限定 `content/posts/3DGenAI` / `content/posts/3dgenai`,不动其他重影路径(spec 风险)
- 绝不 `git checkout` 大写重影路径(会用旧 blob 覆盖磁盘真实文件,踩坑记录)
- 脚手架全删:prompts / outline.md / batch.json / README.md / 3DGenAIimg / .DS_Store(spec 决策)
- 图片用 pngquant 量化,保持原尺寸 1376×768(spec 决策)
- 不改六部曲正文文字内容,只做归位(spec 不在范围内)

- 脚手架:39 个 prompts 条目、6 个 outline.md、1 个 batch.json、1 个 README.md、空 3DGenAIimg/、.DS_Store

### Task 3: pngquant 压缩图片

**Files:**
- Modify(原地压缩): `content/posts/3dgenai/**/*.png`(38 张)

**Interfaces:**

- [ ] **Step 1: 批量压缩(原地覆盖)**

`--ext .png --force` 让 pngquant 覆盖原文件;`--skip-if-larger` 防止反而变大;`--quality=65-85` 平衡质量。

```bash
cd /Users/ethanhu/Documents/self/my_blog
find content/posts/3dgenai -name '*.png' -print0 | \
  xargs -0 pngquant --quality=65-85 --skip-if-larger --ext .png --force --
echo "exit: $?"
```
Expected: `exit: 0`(个别图若已优化触发 skip-if-larger 返回非 0 也可接受,关键看下一步尺寸与体积)
